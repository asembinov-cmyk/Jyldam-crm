Приём заказов в Jyldam Logistics

Подключите интернет-магазин, CRM или 1С и отправляйте заказы напрямую к нам. Ниже — полный протокол: авторизация, поля, ответы, примеры и типичные ошибки.

Интеграция работает по схеме сервер → сервер. Ключ доступа не должен попадать в браузер покупателя.

Обзор

Заказ создаётся одним запросом POST на эндпоинт выше. Тело — JSON с данными получателя и составом заказа. Для доступа нужен ключ, который выдаёт менеджер Jyldam Logistics.

Ответ всегда приходит в формате JSON в обёртке result.

Чек-лист подключения партнёра

  1. Получить у менеджера Jyldam Logistics ключ доступа и убедиться, что карточка партнёра активна.
  2. Реализовать вызов только со своего сервера (бэкенд магазина, CRM, 1С) — не из браузера покупателя.
  3. Передавать ключ в поле secret (в теле запроса) или в заголовке X-Api-Token.
  4. Отправлять обязательные поля: телефон, страна, состав заказа; желательно — order_id для защиты от дублей.
  5. Обрабатывать ответ: успех success: TRUE с полем id, при ошибке success: FALSE с текстом.
  6. Сохранять возвращаемый id — это внутренний номер заказа в Jyldam Logistics для сверки.

Авторизация

Каждый запрос должен содержать секретный ключ, выданный вам Jyldam Logistics. Ключ передаётся в теле запроса (поле secret) — либо, по желанию, в заголовке X-Api-Token.

Без верного ключа запрос отклоняется:

// ответ при неверном ключе
{ "result": { "success": "FALSE", "message": "Incorrect hash" } }
Ключ выдаётся индивидуально каждому партнёру. Не передавайте его третьим лицам и не размещайте в коде, доступном покупателю.

Формат состава заказа

Поле offer — строка вида {тех.название} - {количество}. Несколько товаров разделяются запятой:

brutalin - 2, oko_plus - 1

Требования к тех.названию товара

— только латиница и цифры;
— без пробелов и спецсимволов;
— тех.названия согласовываются с менеджером заранее и должны совпадать с кодами товаров на складе Jyldam.

Если тех.название не совпадёт с товаром на складе, заказ всё равно примется, но будет помечен как «требует привязки» — оператор сопоставит товар вручную.

Ответ сервера

Успех

{ "result": {
    "success": "TRUE",
    "message": "200 OK",
    "id": "a1b2c3d4-…"
} }

id — идентификатор заказа в Jyldam Logistics. Сохраните его для сверки.

Ошибка

{ "result": { "success": "FALSE", "message": "phone is required" } }

Примеры

PHP (cURL)

$secret = "ВАШ_КЛЮЧ"; // выдаётся Jyldam Logistics

$data = [
    "secret"      => $secret,
    "phone"       => "77017777777",
    "country"     => "kz",
    "offer"       => "brutalin - 2, oko_plus - 1",
    "order_id"    => "123321",
    "name"        => "Иван Иванов",
    "addr"        => "Алматинская обл., Карасайский р-н, Шамалган, Курмангалиева 45",
    "city"        => "Алматы",
    "price"       => "6400",
];

$ch = curl_init();
curl_setopt($ch, CURLOPT_URL, "https://tdexhspesodslwgbxojw.supabase.co/functions/v1/receive-order");
curl_setopt($ch, CURLOPT_RETURNTRANSFER, 1);
curl_setopt($ch, CURLOPT_POST, true);
curl_setopt($ch, CURLOPT_POSTFIELDS, ["data" => json_encode($data)]);
$result = curl_exec($ch);

curl (JSON)

curl -X POST \
  "https://tdexhspesodslwgbxojw.supabase.co/functions/v1/receive-order" \
  -H "Content-Type: application/json" \
  -d '{
    "secret": "ВАШ_КЛЮЧ",
    "phone": "77017777777",
    "country": "kz",
    "offer": "brutalin - 2",
    "order_id": "123321",
    "name": "Иван Иванов",
    "city": "Алматы",
    "price": "6400"
  }'

Повторы и обновления

Если прислать заказ с тем же order_id, что и ранее, заказ не дублируется — его данные обновляются, а id в ответе остаётся прежним.

Так можно присылать обновления по уже переданному заказу: изменившиеся статусы (send_status, status_kz) или трек-код (kz_code).

Безопасность

— Вызывайте API только со своего сервера. Ключ не должен попадать в код страницы, доступной покупателю.

— Передавайте данные по HTTPS.

— При подозрении на утечку ключа сообщите менеджеру Jyldam Logistics — вам выдадут новый.
